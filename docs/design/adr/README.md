## 0.7.0 — Render pipeline
- [0031](0031-parameterized-views-free-variables-and-field-extraction.md) — Parameterized views: free variables + a bindings environment. **Rewritten 2026-07-09 (forward model):** a view **predicates** (`Filter`) or **projects** (`field_of`) forward from the side that carries the field, over a full-field roster (0034); the Filter value slot is field-typed (`entity_ref` → an entity operand compared by id), parameterizable, with cardinality-driven `overlap` (op enum 6→4); `field_of`'s output kind follows the projected field (ref → nodes, value → values); field selectors = **intersection of fields over the input set**; referenced-by = a computed `'References'` node-set field, not an operator. **Withdraws Match (0033); the two-pipe / id-extraction revision is superseded.** (#184; amends 0018/0020/0025/0027, builds on 0023, depends on 0034)
- [0032](0032-parameter-declaration-and-binding-provenance.md) — Parameter declaration + binding provenance. **Amended 2026-07-09:** **promote-in-place** is primary — a runtime formal is a promoted Filter value slot, named in the view's parameter list, its type **derived from the field** (intersection when shared across slots); the #182 wrapper renders one strip control per formal. `$self` = a reserved **wired source node** (ambient, no control, type = anchor kind), not a formal. Binding ladder: surface/reserved `$self` / author default (inline literal) / runtime strip / context-wire (v2); runtime picks ephemeral (view = template). The **declared Parameter-node-typed-by-reference is deferred** to node-set-source params (no current use case). **Amended 2026-07-19 (Amendment 2, #199): `$self` REMOVED entirely — no render surface bound it; reintroduce with the anchored surface. Every ADR's `$self` mentions are superseded by Amendment 2.** (#184; follows 0031, builds on 0023/0022/0026)
- [0033](0033-match-flat-relational-sibling-of-nest.md) — Match: the flat relational sibling of Nest. **WITHDRAWN 2026-07-09** — superseded by 0031's forward model: forward predication keeps the subject (no join to recover it) and `field_of` projects, so reference matching needs no dedicated operator; any-field referenced-by is the `'References'` computed field; flat-vs-tree is presentation (0027/#181), not grammar. Kept as the record of why the operator is unnecessary. (#184)
- [0034](0034-view-universe-is-a-full-field-roster.md) — The view universe is a full-field roster: the evaluator can only reference fields present in the roster it is handed, so the forward model (0031) requires every node's complete field set; the thin manuscript-structure projection (computed counters + surfaced status/color) is insufficient. Draft membership either enriches the roster or (leaning) evaluates over the full node index and projects the id-set onto the structure tree. (#184 prerequisite; relates #182/#112; follows 0025/0031)
- [0035](0035-viewnodelist-viewresult-is-the-sole-input.md) — `ViewNodeList` (the #182 render wrapper, so named): a view's output is **exactly one `ViewResult<T>`, always** (`T` a node — 0031's value-set payload is an eval-internal operand, never reaches the wrapper), and that is `ViewNodeList`'s **sole input** — no `ViewResult[]`, no `ViewResult | T[]` union. Non-view sites lift via a `ViewResult` constructor (`nodeSet(nodes)` = degenerate one-stream; optional `concat(...)` mirrors 0027 §D for hand-assembled results — not grammar, same-kind). Rule (0027 §E / 0019 / 0028): `nodes` = membership (deduped), `groups` = presentation (may repeat). `ViewNodeList` **composes** NodeList, does not extend it; absorbs `ViewGroupedList`+`GroupTree`. (#182; follows 0021/0022/0025/**0027**/0034; consistent w/ 0019/0028/0031 §D; relates #112/#181/#184)
- [0036](0036-implicit-explicit-views-and-empty-evaluation.md) — **The empty/null view evaluates to the empty set** (kills the `expr:null → whole roster` hack at `evaluateView.ts:329`/`:668`/`:686`; distinct from the neutral-reset for an unset *combinator operand* at `:1033`, which stays). ## 0.7.0 — Project hierarchies (#7)
- [0039](0039-project-hierarchies-inheritance-affordances-overrides.md) — Project hierarchies: a project = any folder with `project.yaml`; `project.md` = the singleton project node; **Amendment 1: inheritance is DECLARED, not inferred** — the filesystem walk enumerates candidates (finite, cycle-free by construction) and the project records which it inherits from (a transitive `parent:` link was rejected: user-editable ⇒ cycles); N levels, no hardcoded tier names, scenes stay book-scoped; ancestor-only visibility; level-agnostic opening; **inheritance is virtual membership** — ancestor nodes are materialized into the open project, so a per-field override folds into *base* (at materialization) and never enters the position-keyed `effective_state` chain; three edit affordances (direct edit / override-keeps-inheriting / fork-severs-it, `⧉`, keeps the id); layer overrides are `overrides/` **deltas on the mutation structure** (set/add/remove, `target: <id>`, salted id, descendant-wins per item) — NOT sparse lore nodes; provenance must be visibly marked **Amendment 2: ONE traversal** — six derivations of the root→base chain collapse into the promoted `IndexLayer`, and the root is *stipulated*, not inferred from a stray `metadata.schema.yaml` (#329) (**Accepted** 2026-07-19, PR #319; #7; companion 0040; amends 0013; gesture → 0042; mark → #304; **does not** amend 0005)
- [0040](0040-node-index-persisted-incremental-no-sqlite.md) — The node index is a **persisted, incrementally-maintained id-map** (`id → [candidates by layer]`, each `path/kind/entry_type/title/layer rank`) plus field-qualified edges served by a **reverse adjacency map**; bodies lazy; JSON snapshot under `.cache/`, **no SQLite** (measured — memory never the constraint, parsing is the cost, and a flat edge list is the one option that loses to SQLite); staleness by manifest diff; incremental + change-gated; concurrent writers an explicit non-goal; prerequisite for #7, which it also *materializes* (**Accepted** 2026-07-19, PR #319; #7; companion 0039; follows 0025)
- [0042](0042-inherited-node-edit-gesture.md) — **The edit gesture on an inherited node: two bounded axes, one rule.** A **layer picker** docked in the editor *header* (not the rail — the rail collapses, and body/title write to L too) is the hierarchy axis's counterpart to 0013's foot-docked manuscript scrubber, and *position is the mode* on both. The **authoring layer L is bounded** (`node's owning layer ≤ L ≤ open project`): at the owning layer an ordinary write, below it an `overrides/` delta — so direct-edit and override are **one write at two positions**, and only `⧉` fork stays a button. The bound is on the **write target only**; every picker still resolves `base → L`, so truncation drops the layers *below* L (whence "a name is asserted at or above the layer holding the value that uses it", free). The axes form an **L, not a grid** — an ancestor has no manuscript, so stops ≥ 1 exist only at the open project. **Mutation edit-in-place = a stop is a unit** (0016): editing at stop N edits that unit's row, or adds one. There is **no "between"** — the scrub coordinate is an integer index over units, not a continuum, so an intermediate position is not expressible (and must not be made so: it would need a prose cursor the card deliberately lacks, 0003/0001). The card edits the units that exist; it never creates positions. One rule covers all four writes — *a picker position names a container; the edit writes into it; every container reachable at L belongs to L* — which is also what keeps #66 shut (exactly one target file, named before the write). Deltas are authored by 0017's list-edit rules. **The bar has no save-time gate — lore is autosave-only (6 s debounce, no Save button)**, so it is: a confirm **on entry** (never `dontShowAgainKey`), a `--warn` header strip **naming the target in words** while there, a layer-aware footer echo (*"Saved to series"*) as the only per-write signal autosave permits, and **non-sticky L** (resets on pane close / entry switch). Settles 0013 Amendment 1 (stop-0 editing on an inherited entry may ship); does **not** overturn 0039's descendant-wins composition; read-side layer scrubbing stays deferred to Views 2.0. Rejected: navigation-only (the lazy edit still happens, and is then indistinguishable from a deliberate divergence), per-field control triplets, a scalar-fields-only carve-out (partial editability surprises only when it bites), deriving the target from the node's owning layer. (**Accepted** 2026-07-21, PR #351; #308; gates #313/#314; prerequisite #339 landed in PR #341)

## 0.8.0 — Remaining features
- [0043](0043-scene-snapshots-are-witnesses.md) — **Scene snapshots are witnesses**: a fully correct snapshot of a scene is **not constructible** (mutation intervals span scenes — a scene inside an interval it does not own carries no markers yet its meaning is set elsewhere; manuscript order lives in another file; ADR-0039 adds a second drift axis through the layer chain), so the feature accepts that instead of defeating it — **prose is restored byte-exact, context is captured and only reported when it has drifted**. Witness = `effective_state` output + a per-entity change token (#314's composite revision is the inheritance-axis detector); diagnostic only, never restored, never read by the resolver, preserving the derived-never-stored rule (0010). **Scenes only in v1** (lore entries are the drift *targets* and become composed values under 0039). **Full copies, not deltas** — plain text buys the diff *view*, it does not oblige delta *storage*. Two triggers with asymmetric retention: lazy session-boundary capture (materialized only if the session dirties the doc, **keep last 5** — the automatic tier is a prosthetic for the author's own recall, and that is roughly the depth a person holds) + an explicit Scrivener-style button (**never thinned**). **Identity:** a snapshot carries its **own id**, never the source's (the front-matter id is canonical and 0040's index maps `id → candidates by layer` — reusing it is an index collision); stored as **two files**, a byte-for-byte copy of the scene plus a sidecar (`id`, `snapshot_of`, `captured_at`, `retention: thinned|kept`, capture-time `schema_version`, the witness), so byte-exactness is *proven by file copy, not reconstructed through a serializer*. **The directory layout `snapshots/<source-id>/` IS the lookup table** — "how many/which" is a listing, no index to invalidate (an index only when a query the filesystem can't answer cheaply exists); deliberately makes a *directory* name load-bearing where filenames are cosmetic, since ids are stable and titles are not. **Invariant: `snapshots/` is excluded from the node index once, at the index** — not filtered per consumer, or every scene view gains 5× members. **Deleting a scene deletes its snapshots**, count named in the confirmation — an earlier draft retained them, which was wrong because **unreachable data is not preserved data**: the directory is named by id, the author knows scenes by title, and with no node left to hang an affordance on, retention without a recovery surface is disk usage plus a false sense of safety (the worst of the three options, being the one that *looks* safe). Recovery is out of scope — it belongs to a general undelete, not one per feature — and deleting also keeps snapshots from becoming the only soft-delete-ish thing in an app whose deletes are all `path.unlink()`. Durable property either way: **a scene and its snapshots are one unit of deletion**, which would still hold under a future trash metaphor (whether snapshots participate is that design's call, not committed here). Names the two live collisions the feature must dodge (`revision` = the transient sha256 concurrency token; "time travel" = 0013's in-fiction scrubber). Rejected: per-save snapshots (autosave's 6 s debounce makes the boundary meaningless), git-style history (no git anywhere in the codebase), whole-project checkpoints (self-consistent but makes losing unrelated work the default failure mode). **Drift reporting is advisory, never blocking** — a tool, not a straitjacket; the binding obligation is that the report name the *specific* change in the author's vocabulary, since it is then the only protection (a vague warning trains dismissal). **Not blocked on #314**: `revision` is consumed as an opaque change token, so 0039's composite hash improves the detector with no change here (pre-#314 tokens report the inheritance axis as *unknown*, not *unchanged*). Sequencing is governed by **file contention, not dependency** — backend near-disjoint, surface should follow the 0042 editor rework. User-facing surface deliberately **not** designed here. **The third drift axis is value *reinterpretation*, not schema change** — invisible to both other detectors (`_revision` hashes the entity file; the schema is a separate merged file), but far narrower than "the schema changed": stored files are **sparse**, so an added field is simply absent and a deleted one is stripped, needing no detector. Only a type change or a shrunk option set makes surviving bytes mean something new, so the token covers the type/constraints of **the fields the witness recorded**, not the merged schema (a whole-schema hash was drafted and rejected: it fires on harmless edits, and under an advisory model a detector that cries wolf trains the dismissal that makes the report worthless — **detector precision is part of the report-quality obligation**). **Snapshots are irreplaceable-but-not-authoritative** — a category the project has not had: unlike `.cache/` (rebuildable) and `.migration-backups/` (disposable), the pre-1.0 "bump the key and rebuild" escape hatch does **not** apply, so a pre-1.0 store must stay readable after 1.0. **Snapshots are immutable — migration happens at restore, not at rest** (rewriting a witness destroys it), which yields the free result that they need no migration backup, and imposes a **hard pre-1.0 requirement on the migration framework**: content migrations must be expressible as per-document functions, which `MigrationFn = Callable[[Path], None]` (`migrations.py:44`) cannot express today. (**Draft — awaiting red pen**; #6; relates #314; follows 0001/0002/0003/0010/0039; constrains `services/migrations.py` before 1.0)
