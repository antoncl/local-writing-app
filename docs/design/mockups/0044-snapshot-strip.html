At rest vs. engaged

The same strip in both states, static, so the difference in weight is visible without leaving compare mode. Writing happens against the first one nearly all of the time.

What this is testing

Position is the mode

Parked at Live the only action is Snapshot. Click a notch and the strip offers Restore · Pin · Delete plus the compare toggle. Same rule ADR-0013 uses on the mutation scrubber — no context menus, nothing hidden behind a right-click. The strip's size follows the same rule, not just its contents.

Notches, not beads

The mutation scrubber uses round stops on a rail. These cut into the edge, so if both axes ever appear on one card they read apart at a glance while the gesture stays identical. Tall + filled = explicit (kept). Short + faint = automatic (thinned, last five).

Time runs left → right

Oldest at the left, Live at the right end — deliberately not the mutation scrubber's home-at-left, because there base is earliest. Copying its home position would make the two strips look alike while reading backwards.

One colour rule

The colour says which version the text belongs to. An addition is “now” because it exists only in the current scene; a deletion is “snapshot” because it exists only there; a modification is simply the two adjacent. Three cases that could drift apart become one that can't.

Inline vs stacked is structural

A change inside one block renders side by side; a change spanning blocks stacks. No length threshold — a word count deciding the layout makes it jitter as you type. Paragraph 2 is inline, paragraph 4 is stacked.

Flip, don't split — and keep the tint while you flip

No side-by-side panes to keep in sync. The same region swaps between the two states — the eye is poor at parallel scanning and excellent at change detection.

The changed regions stay tinted in every compare state, not only Both. An earlier pass rendered the single states as unmarked prose, which broke the gesture: with nothing marked, a flip changes words somewhere and the eye has no anchor to hold. The tint is the anchor — the words swap underneath a patch that doesn't move. Only Live is unmarked, because there is nothing to compare it to.

Two axes, no held modifier

Holding Shift was wrong: repeated holds trip Windows FilterKeys, and five presses fire Sticky Keys. It isn't needed either — the compare view is read-only (the live TipTap buffer stays mounted and hidden beneath), so the entire unmodified keyboard is free and nothing has to be a chord.

←→ move through time; Active / Snapshot flip the version. Left hand on the letters, right hand on the arrows — and not ↑↓, which fight page scroll on a long scene. Press the active one again for Both, Esc for Live.

Spacing is the timeline — and it can't be linear

Notches sit at their age, so a tight cluster is an afternoon's work and a long run is a week away from the scene. Linear time would break instantly: one snapshot from last week plus four from this morning piles the recent four into an unreadable clump at the right edge — and those are the ones you reach for. A log scale spreads recent history and compresses deep history, which is also how people remember their own work. Faint 1h / 1d / 1w ticks make the scale legible rather than merely implied, and a minimum gap keeps notches from touching however they bunch.

The track never changes width

Notches are positioned as percentages of the track, so anything beside it that grows or disappears slides every notch along the timeline. Two things did: the “Snapshot · this morning” label appearing on park, and the camera button growing a text label. The label moved down into the actions row — which only exists while parked, so it can be any width — and the camera is now a fixed square in both states. A timeline that moves when you use it can't be read.

Compact at rest

While writing, the strip is a quiet ruled line — small notches, camera only, no labels. Parking on a notch is what earns the taller strip and the controls. Same compact-at-rest, expand-on-engage shape ADR-0038 §A chose for the view-designer nodes.

The camera is an affordance, so it is not an annotation

The design language already rules that a mark is never both — which is why ⧉ is the fork button only. So the camera marks capture, and never appears on a changed field. The general rule that caps the glyph pile-up: a glyph marks what is true about the value; colour marks what is true about the view. ⤳ mutated and ti-versions overridden are permanent facts about where a value came from. A snapshot difference is not a property of the field at all — it exists only while you are parked, and vanishes at Live. Lenses get colour, not glyphs.

Fields flip only

The rail never interleaves — a value is atomic, so it resolves in one blink. Same colours, same meaning, no second vocabulary. Watch Status and POV as you flip.

Open: the description

Hover a notch. Two carry a one-liner, three don't — the empty case is the common case, so the tooltip has to read well without one. Whether the description replaces the date or sits under it is exactly what this is for.

The colours are the open question

Almost every hue is spoken for: teal is the accent, violet is the mutation axis, amber is --warn, brick red is --danger (reserved for destructive — and this isn't an error), slate-blue is lore. Red/green is out on both accessibility and semantics: this is before and after, not bad and good.

Every candidate reads warm = live, cool = archived — the past reading cooler is an intuition most people already have, and it puts the pair on an axis rather than making them arbitrary. Use the bench at the top of the page to put each one behind the real prose.

--diff-wascool · snapshot

Three things the theory can settle, leaving the rest to the eye:

  • Equal chroma. If one tint is more saturated than the other it reads as more important, and the pair stops meaning then vs. now and starts meaning significant vs. incidental. Every candidate is balanced.
  • Light needs to be lighter than dark is dark. A wash that reads on #1f2522 is usually too faint on white — dark surfaces tolerate a smaller lightness step. Each pair is tuned separately per theme, not inverted.
  • Hue cannot be the only channel. Roughly 1 in 12 men has a colour-vision deficiency, and the warm/cool axis is exactly the one most affected. Tick strip the hue to see each pair in greyscale: if the two are still distinguishable, the lightness step is carrying enough on its own. That is also why every tint has a darker edge rule under it — an underline survives what a wash does not.

What the theory can't settle: whether a given pair is restful behind prose you're reading for an hour. Toggle the theme too — a pair can be right in one and wrong in the other.
