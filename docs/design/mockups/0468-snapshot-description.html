The snapshot description — three places it could live

The sidecar gains an optional one-line description (ADR-0043 Amendment 1). §L already fixes that it augments the date and that the empty case is the common case. What is open is the edit surface. Pick the row that reads best while parked — with a description and, more importantly, without one — in both themes.

The Salt Line

The tide went out further than Maren had ever seen it, and kept going. Where the harbour should have been there was only ribbed sand, black with weed, and the boats lying over on their sides like something sleeping.

…parked on a snapshot — the body above is the read-only overlay. This mockup is only about the row below.

What each row is trying to be

A · an inline field in the actions row

A quiet one-line input sits in the row, between the date and the controls. Empty, it shows a faint Add a description… placeholder; it is always there, so it is discoverable without a gesture. Cost: it is always there — a field the author mostly leaves blank, taking row width the A/S/B control and Restore also want. Watch the narrow-pane case.

B · behind a pencil

At rest the row shows only what exists: the date, and the description after it when there is one. A small ✎ edits it; with none, a faint + describe appears. Nothing empty is ever shown. Cost: writing a description is a click behind an affordance the author has to notice — the enrichment is one step less discoverable.

C · a second line under the date

The row's left cell stacks: date on top, description beneath it, full-width, with the controls to the right unchanged. Groups the description with the when rather than the verbs, and gives a one-liner room to be read. Cost: the row is always two lines tall even when the description is blank, unless the empty line collapses — which is what the empty toggle tests.

All three keep §L's contract: the date line survives, and the empty state is never an empty box begging to be filled. The tooltip carries the description too (as in 0044-snapshot-strip.html); this is only about the parked row.
